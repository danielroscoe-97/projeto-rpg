## 3. O que precisa QA manual (user-facing)

### 3.1 AC-3c-04 — Undo toast em unlink de NPC chip (commit `6e537b2c`)

**Onde:** `/app/campaigns/[id]?section=notes` — as notas do DM que tenham chips de NPC linkados.

**Como testar:**

1. Entrar em uma campanha como DM
2. Abrir uma nota que já tenha NPCs linkados (chip com nome do NPC)
3. Clicar no "X" do chip do NPC
4. **Esperado imediato (<100ms):** chip some da UI + aparece toast no rodapé "Vínculo removido · Desfazer"
5. **Dentro de 5s**, clicar em "Desfazer"
6. **Esperado:** chip volta, toast desaparece, sem round-trip de rede visível
7. Repetir: remover 3 chips em sequência rápida (<5s entre eles)
8. **Esperado:** toast consolidado "3 vínculos removidos · Desfazer tudo"
9. Esperar 5s sem clicar Desfazer
10. **Esperado:** toast some silencioso, reload da página mostra que os vínculos realmente sumiram do DB

**Regressões a observar:**

- Toast não aparece → i18n `entity_graph.undo_unlink*` não carregando
- "Desfazer" clica mas chip não volta → `onUndo` closure capturou state errado
- Fechar a aba durante janela de 5s sem commit — hoje o `pagehide` tenta flush best-effort. Se o edge não for deletado no DB, é bug crítico (dual-state, UI acha que apagou, DB tem)

### 3.2 Chip navigation (commit `010b1c1e` + post-review `93a89897`)

**Onde:** Notas de player visíveis pro DM (`/app/campaigns/[id]?section=notes`, panel "Notas dos jogadores").

**Pré-requisito:** ter uma nota de player com @mention de algum NPC/Location/Faction/Quest. Se a nota não tiver mentions, não aparece chip. Pra seed rápido, usar `/api/e2e/seed-note-mentions` ou inserir via Supabase SQL editor.

**Como testar (4 fluxos):**

1. **Chip de NPC:**
   - Clicar no chip de NPC na nota
   - **Esperado:** URL vira `?section=npcs&npcId=<uuid>`
   - **Esperado:** tab de NPCs abre, card do NPC está expandido, scroll posiciona ele no centro da viewport

2. **Chip de Location (caso especial — ancestor uncollapse):**
   - Criar uma location filha de outra (ex: "Taverna do Pêndulo" dentro de "Cidade do Pêndulo")
   - Colapsar o parent na tree view
   - Clicar no chip da location filha numa nota
   - **Esperado:** parent descolapsa automaticamente, card filha aparece expandido, scroll centraliza

3. **Chip de Faction:**
   - Clicar no chip de Faction na nota
   - **Esperado:** URL vira `?section=factions&factionId=<uuid>`, card expandido + scroll

4. **Chip de Quest (caso especial — filter reset):**
   - Na tab de Quests, filtrar por "Ativas"
   - Voltar pra notes, clicar no chip de um Quest "Completed" (que está fora do filtro atual)
   - **Esperado:** filtro reseta pra "Todos" automaticamente, card do quest completed aparece, scroll centraliza

**Regressões a observar:**

- Click no chip abre a tab mas card não expande → `focusToken` prop não chegando no card
- Click repetido no mesmo chip não re-scrolla → `handled-ref` guard com bug na identidade do `searchParams`
- Em LocationList tree view, focused card fica invisível → ancestor uncollapse walk falhou
- Card expande mas scroll para no topo → double-RAF timing; pode ser flake
