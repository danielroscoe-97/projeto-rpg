Análise

Gaps Críticos — Priorização

Os 6 gaps mais impactantes identificados na análise dos fluxos, ordenados por risco de churn.

🔴 Crítico — UC-05

Token do Invite Perdido no Auth

Jogador clica em /invite/[token], é redirecionado para login, e o token não sobrevive ao fluxo de auth. Cai no dashboard vazio sem entender o que fazer.

🔴 Crítico — UC-08 (Novo)

Recap sem CTA de Join Campanha

Jogador anônimo termina combate no recap, mas não existe CTA para entrar na campanha. O maior momento de conversão orgânica do app está inexplorado.

🟠 Alto — UC-02

Onboarding Pede HP/AC Prematuro

Step 2 do wizard pede HP e AC dos jogadores, mas o mestre muitas vezes ainda não tem esses dados. Precisa de opção de "convidar por email" e skip.

🟠 Alto — UC-06

Jogador sem Campanha = Tela Morta

"Waiting for invite" não tem CTA. O jogador que cria conta antes de receber um convite fica parado sem saber o que fazer. Precisa de onboarding próprio para jogadores.

🟡 Médio — UC-02

Role Selection Desconectada

A página /app/onboarding/role com RoleSelectionCards existe no código mas não está integrada no fluxo principal do wizard. O usuário não é perguntado sobre seu role de forma clara.

✅ Resolvido — Pós-Onboarding

Tour do Dashboard — Implementado

DashboardTourProvider implementado com 7 steps (welcome, sidebar, quick actions, campanhas, combates, soundboard, complete). Detecta role DM/player e filtra steps irrelevantes. Persiste conclusão no DB.

✅ Aplicado — 2026-04-05

Correções de Playtest — Sprint UC-07 / UC-08

7 bugs corrigidos no fluxo do jogador anônimo (UC-07) após sessão de playtest. Todos commitados em a0839e0 e 05b81dc.

✅ B1 — UC-07 / spells

Popup de magias carregava infinito

SrdInitializer não era montado na rota /join/[token], apenas no layout autenticado. Adicionado na page.tsx do join.

✅ B2 — UC-07 / i18n

conditions.blessed aparecia como chave bruta

6 condições benéficas (blessed, flying, haste, heroism, inspired, raging) não tinham tradução. Adicionadas em EN + PT-BR.

✅ B3 — UC-07 / state

Condições sumiam após ~30s

Polling fetchFullState sobrescrevia estado broadcast. Corrigido com conditionOptimisticRef (guard 5s após mudança broadcast).

✅ B4 — DM / z-index

Botão fechar do log bloqueado pela navbar

Navbar z-50 sobrepunha o painel do log (também z-50). Painel elevado para z-[60].

✅ B5 — DM+Player / UI

Indicador de criatura crítica muito agressivo

animate-pulse (50% variação, 2s) substituído por animate-critical-glow (18% variação, 3.5s). Guard prefers-reduced-motion adicionado.

✅ B6 — UC-07 / UC-08 recap

Combat Recap não aparecia para jogadores

DM só broadcastava session:combat_stats. Adicionado broadcast de session:combat_recap com o CombatReport completo (awards + narrativas + display_name anti-metagaming).

✅ B7 — UC-07 / death saves

Death saves sumiam após polling

Campo death_saves ausente no SELECT da API de estado. Guard otimista comparava contra undefined → sempre falhava. Adicionado ao SELECT e à interface RawCombatantRow.
